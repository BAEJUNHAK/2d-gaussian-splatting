## 1-2. 데이터 구조

### dataset.zip (RGB + Camera) — ~1.23 GB

```
dataset.zip
└── dataset/                    ← 루트 폴더
    ├── rgb_0000.png            ← RGB 이미지 (641개)
    ├── rgb_0001.png
    ├── ...
    ├── rgb_0640.png
    ├── calib_0000.ini          ← 카메라 캘리브레이션 (641개)
    ├── calib_0001.ini
    ├── ...
    └── calib_0640.ini
```

- **총 1,283 파일** (RGB 641 + Calib 641 + 루트 폴더 1)
- RGB 파일은 모두 **1,924,497 bytes** (동일 크기 — Blender 렌더링 특성)
- Calib 파일은 457~461 bytes (뷰별 extrinsic 값 길이 차이)
- 인덱스 0000~0640, 갭 없이 연속
- Calib 파일에 주석 행 없음 (`[camera_0]`부터 시작)

### depth_data.zip (Depth + Camera) — ~37.8 MB

```
depth_data.zip
└── dataset_depth/              ← 루트 폴더
    ├── depth_raw_0000.png      ← Depth 이미지 (641개)
    ├── depth_raw_0001.png
    ├── ...
    ├── depth_raw_0640.png
    ├── calib_0000.ini          ← 카메라 캘리브레이션 (641개)
    ├── calib_0001.ini
    ├── ...
    └── calib_0640.ini
```

- **총 1,283 파일** (Depth 641 + Calib 641 + 루트 폴더 1)
- Depth 파일 크기는 **25~69 KB** (뷰마다 다름 — 전경 형태에 따라 PNG 압축률 변동)
- Calib 파일은 513~517 bytes (RGB zip보다 약간 큼)
- 인덱스 0000~0640, RGB zip과 **완전 일치**

### 두 zip 간 관계

| 비교 항목 | 결과 |
|----------|------|
| 인덱스 매칭 | **641개 전부 완전 일치** (0000~0640) |
| Calib 내용 | **동일** (extrinsic/intrinsic 값 일치) |
| Calib 차이 | Depth zip에만 `# Camera calibration file generated by 'blender script'` 주석 행 있음 |
| 루트 폴더명 | RGB: `dataset/`, Depth: `dataset_depth/` |

두 zip의 calib 파일은 같은 카메라 파라미터를 담고 있으므로, **어느 쪽 calib을 사용해도 동일한 결과**를 얻음.

### 파일 상세 설명

#### `rgb_XXXX.png` — RGB 이미지

| 항목 | 값 |
|------|------|
| 해상도 | 800 x 800 |
| 채널 | 3ch (RGB), 8bit per channel |
| 내용 | Blender 렌더링된 오브젝트 + 회색 배경 |
| 파일 크기 | 1.92 MB (전부 동일) |

#### `depth_raw_XXXX.png` — Depth 이미지

| 항목 | 값 |
|------|------|
| 해상도 | 800 x 800 |
| 채널 | 1ch, **uint16** (0~65535) |
| 배경 | 0 (depth 없음 = 배경) |
| 전경 | raw 값 × 0.01 = 미터 단위 거리 |
| 파일 크기 | 25~69 KB (뷰마다 다름) |

#### `calib_XXXX.ini` — 카메라 캘리브레이션

```ini
# Camera calibration file generated by 'blender script'   ← depth zip에만 있음
[camera_0]
k_matrix=Matrix:3:3:fx,0,cx,0,fy,cy,0,0,1               ← 3x3 intrinsic matrix
r_matrix=Matrix:3:3:r00,r01,r02,...,r22                   ← 3x3 rotation matrix
t_vector=Vector:3:tx,ty,tz                                ← 3x1 translation vector
width=800
height=800
dist_coeff=Vector:5:0,0,0,0,0                             ← 렌즈 왜곡 (항상 0)
clip_start=0.100000
clip_end=1000.000000
```

| 필드 | 설명 |
|------|------|
| `k_matrix` | 3x3 intrinsic matrix (fx, fy, cx, cy). 모든 뷰 동일 |
| `r_matrix` | 3x3 회전 행렬 (world → camera). 뷰마다 다름 |
| `t_vector` | 3x1 이동 벡터 (world → camera). 뷰마다 다름 |
| `width`, `height` | 이미지 크기 (항상 800x800) |
| `dist_coeff` | 렌즈 왜곡 계수 (항상 0 — Blender PINHOLE) |
| `clip_start`, `clip_end` | Blender 렌더링 클리핑 범위 |

카메라 월드 좌표 복원: `cam_pos = -R^T @ T`

## 6. Blender 합성 데이터의 특수성

이 데이터는 Blender로 생성된 합성 데이터이며, 실제 촬영 데이터와 다음과 같은 본질적 차이가 있음:

| 특성 | 합성 (이 데이터) | 실제 촬영 |
|------|-----------------|----------|
| GT Depth | 완벽, 노이즈 없음 | 센서 노이즈, 구멍, 정합 오류 |
| 전경 마스크 | 완벽 (depth > 0) | 불완전, 경계 번짐 |
| 카메라 intrinsic | 모든 뷰 동일, 왜곡 없음 | 뷰마다 다를 수 있음, 렌즈 왜곡 |
| 카메라 extrinsic | 완벽한 구면 배치 | 불규칙, 포즈 추정 오차 |
| 조명 | 균일/고정 | 변동 가능 |
| 텍스처 | 단색 (무채색 회색) | 일반적으로 더 풍부 |

## 7. 자기 가려짐 (Self-Occlusion) 분석

6개 피라미드 사이의 **골짜기(valley) 영역**은 낮은 elevation 뷰에서 가려짐:
- Elevation 0~20도: 피라미드 사이 바닥면이 가려질 수 있음
- Elevation 40~60도: 위에서 내려다보므로 골짜기도 대부분 관측 가능
- 고각 뷰가 더 많아(~315개/641) 이 문제는 상대적으로 완화됨

다만 **오브젝트 바닥면**은 어떤 뷰에서도 관측되지 않으므로 (elevation 범위 0~60도), 바닥면 재구성은 원천적으로 불가.
